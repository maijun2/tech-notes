AWS Blocks とは何か — バックエンド構築フレームワークの概要

AWS Blocks は 2026年6月16日にパブリックプレビューとして公開されたオープンソースの TypeScript バックエンドフレームワークです。各「Block」はアプリケーションコード・ローカル開発セットアップ・インフラ定義を一体化した自己完結型のバックエンド機能単位であり、同一コードが変更なしでローカル環境と AWS 本番環境の両方で動作します。AWS アカウントなしでローカル実行でき、デプロイ時はインフラ学習なしで AWS 上のマネージドサービスへ展開できます。Blocks 自体に利用料金は発生せず、裏側で使う AWS サービスのコストのみを負担します。

背景 — なぜ AWS Blocks が登場したのか

従来のバックエンド開発では、AWS サービスの選定・IaC 記述・ローカル模擬環境の用意という 3 つの負担が重なっていました。Amplify はフロントエンド開発者向けにインフラを隠蔽しましたが、詳細なカスタマイズが難しいという課題がありました。AWS Blocks は「インフラをコードに表現しつつ、ローカル実行をファーストクラスにする」アプローチをとります。AI コーディングエージェントによるコード生成を前提とした設計も大きな特徴で、エージェントが一セッションでデータベース・認証・AI エージェント・ファイルアップロード・バックグラウンドジョブを組み合わせられるよう設計されています。

Block の概念

1 つの Block は以下の 3 要素をひとまとめにしたモジュールです。

  • アプリケーションコード — TypeScript で記述したビジネスロジック
  • ローカル開発セットアップ — Docker や外部サービス不要の完全インメモリ/ファイルシステム実装
  • インフラ定義 — 本番 AWS へデプロイするための AWS CDK ベースの構成

Block 同士は自由に組み合わせ可能で、組み合わせると AWS のベストプラクティスに沿ったインフラが自動構成されます。必要に応じて AWS CDK を直接記述できる「エスケープハッチ」も用意されており、フルマネージドと完全制御の間を行き来できます。

対応フロントエンドフレームワーク

  • SPA: Vite + React
  • SSR: Next.js、Nuxt、Astro

型安全性がエンドツーエンドで保証されており、バックエンドの型変更はフロントエンドの型に自動反映されます（コード生成ステップ不要）。また、Kotlin / Swift / Dart 向けのネイティブクライアントコード生成にも対応しています。

技術者視点でのユースケース

1. SaaS アプリのラピッドプロトタイピング

データベース・認証・ファイルアップロード・バックグラウンドジョブを一セッションで追加し、AWS アカウントなしでローカル完結テストができます。本番移行はコード変更不要で deploy コマンドを実行するだけです。

2. AI コーディングエージェントとの組み合わせ

AWS Blocks は AI エージェントによるコード生成を前提に設計されています。Block を組み合わせるだけでインフラが自動構成されるため、インフラ知識のない LLM でも安全なバックエンドを生成できます。

3. フルスタック型 AI アプリの開発

Agent Block と KnowledgeBase Block を組み合わせることで、ツール呼び出し・会話履歴管理・Human-in-the-loop 承認フローを備えた RAG エージェントを、インフラ設定なしで構築できます。

4. AWS への段階的な習熟

最初は Block を組み合わせるだけで済み、必要になったら CDK エスケープハッチで個別リソースを直接制御できます。Amplify のようにインフラを「隠す」のではなく「透けて見える」ため、AWS の学習素材としても活用できます。

Amplify・CDK との比較

観点 AWS Blocks Amplify Gen 2 AWS CDK
インフラの見せ方 コードに透明に表現 カテゴリ抽象で隠蔽 完全制御（低レベル）
ローカル実行 完全対応（クラウド往復なし） サンドボックス（クラウドが必要） 環境依存
インフラ学習コスト 最小限 ほぼ不要 高い
CDK カスタマイズ エスケープハッチで対応 内部で CDK を使用・一部公開 完全対応
AI コード生成との親和性 高（設計目標） 中 中

始め方

# 新規プロジェクト作成（AWS アカウント不要）
npx @aws-blocks/create-blocks-app

# ローカル起動
npm run dev

# AWS へデプロイ（AWS アカウントが必要）
npm run deploy
