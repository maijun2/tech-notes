Oracle AI Database Private Agent Factory とは

Private Agent Factory は、Oracle AI Database 26ai に統合されたノーコードの「AI エージェント工場」です。ブラウザ上のドラッグ&ドロップ Canvas(Agent Builder)でエージェントやワークフローを設計し、公開すると専用コンテナとして起動して REST API から呼び出せるようになります。

最大の特徴はデータベースネイティブであること。データ・ベクトルストア・MCP ツール・実行ランタイム・監査ログがすべて自社テナンシ内で完結し、企業データを外部へ持ち出さずにエージェント的 AI を実現します。

何を解決するのか

エージェント的 AI(agentic AI)は、多段かつ自律的なワークフローが、リアルタイムで一貫した企業データへアクセスし続けることを前提とします。従来は外部のベクトルストアや API 呼び出しがボトルネック・ガバナンスリスクになりがちでした。Private Agent Factory はデータ移動をなくし、データベース既存のアクセス制御をそのまま使うことで、セキュリティ・ガバナンス・性能を両立させる設計になっています。マルチクラウドおよびオンプレミスの双方に対応します。

アーキテクチャ

設計時(design time)と実行時(runtime)を分離した構成です。作成者はファクトリコンテナが提供するブラウザ UI でフローを組み、その定義は YAML として保存されます。公開(publish)すると、各エージェントが自前のコンテナとして起動し、Oracle AI Database に接続して REST エンドポイントを公開します。

主な構成要素

  • Agent Builder(ノーコード Canvas): MCP サーバ・ドキュメント入力・DB 接続・ユーザーチャット入力・プロンプト・LLM 選択・Select AI ノードをワイヤリングして組む。フローは内部的に YAML 定義化される(Oracle の宣言的言語 Open Agent Spec 系)。
  • Oracle AI Database 26ai: データ基盤であると同時に MCP サーバとしても振る舞う。ベクトルストア・ハイブリッド検索、エージェントの記憶、監査証跡を保持。
  • Select AI 連携: profiles / engines / RAG・SQL ツール / teams を使い、推論を社内データにグラウンディング(in-DB AI 実行)。
  • MCP 統合: MCP でラップできるサービスなら何でもツール化できる(任意 API、CRM、監視ツール、自作マイクロサービス等)。認証は Bearer / API キー / OAuth を UI で設定。
  • プリビルトエージェント: Database Knowledge Agent、Structured Data Analysis Agent、Deep Data Research Agent など(RAG・構造化データ分析・深掘り調査用)を同梱。
  • マルチエージェント・オーケストレーション: スーパーバイザーエージェントがリクエストを分類し、専門サブエージェントへ委譲。コンテキスト受け渡しやエンドポイント管理はファクトリが裏で処理。
  • ランタイム: コンテナは Podman / Podman Compose(rootless)で動作。スケールする本番運用では Kubernetes で管理。LLM・埋め込みモデルは供給コンテナ内でプライベート実行、または公開エンドポイント呼び出しを選べる。

実装(導入)の手順

1. 前提条件を満たす

  • Oracle AI Database 26ai のインスタンスへのアクセス
  • Agent Factory 管理用の専用 DB ユーザー(本番データ用とは分ける)
  • Database Knowledge Assistant を使う場合、ユーザー表領域は BIGFILE かつ 100GB 以上
  • 本番モードでは DB パラメータ max_string_size = EXTENDED
  • LLM エンドポイント(vLLM / OpenAI / Ollama / Google / OCI GenAI などに対応)
  • Podman / Podman Compose(非 root ユーザーで rootless 実行)。Linux / macOS に対応

2. インストールモードを選ぶ

  • Quickstart モード: Oracle AI Database 26ai Free をコンテナでローカル自動構築。DB セットアップ不要ですぐ試せる(UI は既定 8080 番、Free DB は 1521 番ポート)。
  • Production モード: DB と LLM エンドポイントを手動設定し、既存環境に接続。

3. デプロイ

ステージング領域に展開される Makefile でデプロイ一式を管理してコンテナ群を起動します。OCI Marketplace からのインストールも可能です。既定で ポート 8080 の Agent Builder UI にアクセスできます。

# 概念イメージ(実際のコマンド・引数は公式ドキュメントの最新版を参照)
# 1) 配布物を取得し、ステージング領域に展開
# 2) config の DB / LLM エンドポイントを設定
# 3) Makefile 経由でビルド〜起動
make deploy
# → http://localhost:8080 で Agent Builder にアクセス

4. エージェントを構築する

  1. Settings → MCP Servers でエンドポイントと認証を登録すると、ツールが自動でディスカバリされる。
  2. Canvas に MCP Server ノード・DB 接続・ドキュメント入力・LLM・Select AI ノードを配置して接続する。
  3. 複数エージェントは「スーパーバイザー + 専門サブエージェント」構成で組む。

5. テスト → 公開 → 連携

  • Playground で動作をテストする。
  • Publish すると専用コンテナが起動し、セキュア REST API として公開される。
  • 業務アプリからそのエンドポイントを呼び出して組み込む。

まとめ

Private Agent Factory は「ノーコードで素早く作れる」点と「データベースネイティブでガバナンスを担保する」点を両立した、エンタープライズ向けのエージェント基盤です。試すだけなら Quickstart モードで DB 込みのコンテナを立ち上げ、本番では既存の Oracle AI Database 26ai と社内 LLM へ接続する、という段階的な導入がしやすい構成になっています。なお製品は比較的新しく、要件やコマンドは更新されるため、導入前には必ず公式ドキュメント原文(特に Prerequisites / Installation 各ページ)で最新情報を確認してください。
